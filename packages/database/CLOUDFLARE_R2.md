# Cloudflare R2 Configuration

"Make a Sound" uses Cloudflare R2 for scalable, zero-egress audio storage.

## Bucket Structure

- **Bucket Name**: `make-a-sound-assets`
- **Location**: Closest to target users (e.g., `auto` or `us-east-1`)

## Object Naming Convention

`sounds/{user_id}/{sound_uuid}/{timestamp}_{filename}.m4a`

## CORS Policy

For both mobile and web playback, the following CORS policy must be applied:

```json
[
  {
    "AllowedOrigins": ["*"],
    "AllowedMethods": ["GET", "PUT", "HEAD"],
    "AllowedHeaders": ["Content-Type", "Authorization"],
    "MaxAgeSeconds": 3000
  }
]
```

## Signed URLs

- Private sounds (metadata `is_public: false`) must be accessed via signed URLs generated by the Supabase Edge Function `get-sound-url`.
- Public sounds can be accessed via a public R2 custom domain if configured.

## Management

Use the `wrangler` CLI or Cloudflare Dashboard to monitor storage usage and lifecycle rules (e.g., auto-cleanup of non-validated uploads older than 24h).
